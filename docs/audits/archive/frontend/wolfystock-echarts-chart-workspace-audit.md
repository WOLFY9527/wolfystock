## 5. ECharts Import Analysis

Current import style in `DeterministicBacktestChartWorkspace.tsx`:

```tsx
import * as echarts from 'echarts/core';
import { BarChart, LineChart } from 'echarts/charts';
import type { ComposeOption, ECharts, SetOptionOpts } from 'echarts/core';
import { CanvasRenderer } from 'echarts/renderers';
import {
  GridComponent,
  TooltipComponent,
  DataZoomComponent,
} from 'echarts/components';
```

Current registration:

```tsx
echarts.use([LineChart, BarChart, GridComponent, TooltipComponent, DataZoomComponent, CanvasRenderer]);
```

Current renderer:

```tsx
echarts.init(containerRef.current, undefined, { renderer: 'canvas' });
```

Local package evidence:

- `apps/dsa-web/package.json`: `"echarts": "^5.6.0"`
- `apps/dsa-web/package-lock.json`: `echarts` locked at `5.6.0`
- `zrender` locked at `5.6.1`
- local `node_modules/echarts` contains `core.js`, `charts.js`, `components.js`, `renderers.js`, `lib/export/*`, and type exports for `LineChart`, `BarChart`, `GridComponent`, `TooltipComponent`, `DataZoomComponent`, `DatasetComponent`, `CanvasRenderer`, and `SVGRenderer`

Conclusion:

- The current code is already using module-based imports and explicit registration.
- It does not import the full `echarts` entrypoint.
- A later implementation could still try more granular import paths from local ECharts exports, but the package-level export files already point at install modules intended for tree-shaking.
- Required chart features appear to be line series, bar series, cartesian grid/xAxis/yAxis, tooltip with axis pointer/crosshair behavior, inside data zoom, gradients through `echarts.graphic.LinearGradient`, and canvas rendering.
- `DatasetComponent`, `LegendComponent`, `TitleComponent`, `SVGRenderer`, and slider dataZoom are not explicitly required by the current option object, though `DataZoomComponent` currently pulls dataZoom internals including slider sources.
